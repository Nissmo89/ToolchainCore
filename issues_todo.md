# Issues to Fix (Bugs & Compatibility Errors)

Below is the list of compatibility errors and bugs identified in the current implementation. We will resolve these one by one and remove them from the list as they are completed.

- [x] **Issue 1: Hardcoded MSVC Archiver Flags on Windows**
  * **Description**: `ExternalCompilerBackend::compile` checks `#ifdef Q_OS_WIN` and immediately uses MSVC-style archiver arguments (`/NOLOGO /OUT:`), which fails when using MinGW/GCC `ar` on Windows.
  * **Fix**: Parameterize the archiver flags based on the detected archiver type, not the host operating system.

- [x] **Issue 2: Lack of MSVC (`cl.exe`) Discovery**
  * **Description**: The discovery mechanism in `Environment::compilerCandidates` does not look for `cl.exe` on Windows.
  * **Fix**: Add `cl` to the candidates list on Windows, and locate it properly.

- [x] **Issue 3: Lack of MSVC Argument Translation**
  * **Description**: The arguments generated by `ExternalCompilerBackend` are GCC-style (e.g. `-std=`, `-o`, `-I`, `-L`, `-l`), which `cl.exe` rejects.
  * **Fix**: Integrate MSVC argument translator.
